# Dark Factory Coordinator — Shared Artifacts Registry

**Date:** 2026-04-09
**Feature:** dark-factory (coordinator agent extension)
**Phase:** DISCUSS — Phase 2 Journey Design

---

Every `${variable}` referenced in journey steps has a single documented source.

## Pipeline Artifacts

| Artifact | Source | Created At | Consumed By |
|----------|--------|------------|-------------|
| `${run_id}` | `queue_factory_run` response | Queue step | All subsequent MCP calls |
| `${thread_message_id}` | `queue_factory_run` response (set on first ChannelNotifier post) | Queue step | All thread replies |
| `${spec_path}` | Human input to `queue_factory_run` | Queue step | Implementing agent, verification agent |
| `${channel_id}` | Human input to `queue_factory_run` | Queue step | ChannelNotifier |
| `${bot_user_id}` | MCP session authentication | Session start | All list/query operations |
| `${commit_sha}` | `git rev-parse HEAD` by coordinator | Claim step | Stored as `spec_commit_sha` for version pinning |
| `${claim_token}` | Generated by `claim_factory_work` / `claim_verification_work` | Claim step | All mutations (heartbeat, submit, cancel) |
| `${worktree_path}` | Coordinator creates `.factory/run-{id}/` | Spawn step | Implementing agent works here |
| `${agent_id}` | Coordinator's internal tracking | Spawn step | Heartbeat delegation, result collection |
| `${branch_name}` | Convention: `factory/run-{id}` | Pre-deploy step | submit_factory_result, verification agent checkout |

## Clarification Artifacts

| Artifact | Source | Created At | Consumed By |
|----------|--------|------------|-------------|
| `${clarification_id}` | Agent-generated: `{section}:{subsection}:{item}` | Ambiguity detection | Response matching, spec amendments |
| `[CLARIFY:...]` message | Coordinator posts to thread | Clarification request | Human reads and responds |
| `[RE:CLARIFY:...]` reply | Human posts to thread | Human response | Coordinator matches and forwards |
| `[SPEC-AMENDMENT:...]` | Coordinator generates post-run | Run completion | Human approves/rejects |

## Verification Artifacts

| Artifact | Source | Created At | Consumed By |
|----------|--------|------------|-------------|
| `${spec_commit_sha}` | Stored at claim time | Implementation claim | Verification agent reads spec at this version |
| `${scenarios_run}` | Verification agent test execution | Scenario run | submit_verification |
| `${scenarios_passed}` | Verification agent test execution | Scenario run | submit_verification |
| `factory/verify-run-{id}` | Verification agent pushes | Post-scenario | Human review artifact |

## State Artifacts (Coordinator-Local)

| Artifact | Source | Scope | Persisted? |
|----------|--------|-------|------------|
| `active_agents` map | Coordinator internal | Session | No (lost on crash) |
| Clarification tracking | Coordinator internal | Session | No (thread is the record) |
| Agent sub-state (`working`/`clarifying`) | Coordinator internal | Session | No (DB still shows `implementing`) |

**Recovery gap:** All coordinator-local state is lost on crash. The thread messages and DB state are the durable records. Recovery must reconstruct from these.

## Data Flow

```
Human writes spec
  -> ${spec_path} stored in factory_runs.spec_path
    -> ${run_id} generated (Snowflake)
      -> ${commit_sha} captured at claim time
        -> ${claim_token} generated, required for all mutations
          -> ${branch_name} set on successful implementation
            -> ${scenarios_run/passed} set on verification
              -> Terminal state: completed | needs_review | cancelled
```
